# Slack 스타일 메시징 시스템 - 미구현 기능 명세서

본 문서는 10년 차 시니어 개발자 관점에서 정리된 현재 시스템의 미구현 기능 및 향후 구현 계획을 담고 있습니다.

## 1. 개요
현재 핵심 DM 기능(메시지 전송, 편집, 삭제, 리액션, 읽음 표시)은 구현되었으나, 엔터프라이즈급 협업 툴(Slack 등)로 진화하기 위해 필요한 팀, 채널, 알림, 검색 등의 기능이 아직 구현되지 않았습니다.

## 2. 미구현 기능 리스트

### 2.1 팀(Team) 기반 구조 및 관리 (구현 완료 ✅)
- **현황**: 프론트엔드 UI 및 API 연동 완료. 팀 생성, 수정, 삭제, 멤버 초대, 권한 관리 기능 구현됨.
- **필요 기능**:
    - 팀 생성, 편집, 삭제 페이지 및 모달. (완료)
    - 팀 멤버 초대 및 권한(Role) 관리 UI. (완료)
    - 여러 팀을 전환할 수 있는 사이드바 UI. (완료)

### 2.2 채널(Channel) 및 그룹 대화 관리 (구현 완료 ✅)
- **현황**: 프론트엔드 UI 및 API 연동 완료. 공개/비공개 채널 구분, 채널 생성/수정/삭제, 채널 멤버 관리 기능 구현됨.
- **필요 기능**:
    - 사이드바 채널 리스트 (공개/비공개 구분). (완료)
    - 채널 생성 및 설정 (이름 변경, 설명 수정) 모달. (완료)
    - 채널 멤버 리스트 및 관리 UI. (완료)

### 2.3 실시간 알림(Notification) 시스템 (구현 완료 ✅)
- **현황**: 실시간 알림 및 토스트 시스템 구현 완료. 멘션(@) 및 스레드 답글 시 실시간 알림 센터 연동됨.
- **필요 기능**:
    - 새 메시지, 언급(@멘션), 팀 초대 등에 대한 실시간 알림. (완료)
    - 알림 센터 UI (벨 아이콘 및 안 읽은 알림 배지). (완료)
    - 실시간 알림 팝업 (Toast) 기능. (완료)

### 2.4 스레드(Thread/Reply) 기능 (구현 완료 ✅)
- **현황**: 실시간 스레드 시스템 서비스 적용 완료. 메인 채팅과 스레드 전용 패널 간의 실시간 동기화 지원.
- **필요 기능**:
    - 특정 메시지에 대한 답글(Reply) 달기 버튼. (완료)
    - 우측 스레드 전용 패널 표시 및 실시간 업데이트. (완료)
    - 메인 메시지 내 답글 수 표시 및 이동 기능. (완료)

### 2.5 고도화된 검색 (메시지, 파일, 채널, 멤버) (구현 완료 ✅)
- **현황**: 통합 검색 시스템(Command Palette) 구현 완료. 메시지, 채널, 멤버에 대한 탭 기반 검색 지원. 
- **필요 기능**:
    - 전체 메시지 본문 검색. (완료)
    - 참여 가능한 채널 및 멤버 검색. (완료)
    - 실시간 검색 제안 및 필터링 UI (Debounce 적용). (완료)

### 2.6 파일(미디어) 프리뷰 및 업로드 진행률 (구현 완료 ✅)
- **현황**: 실시간 업로드 진행률 표시 및 미디어 프리뷰 시스템 구축 완료.
- **필요 기능**:
    - 파일 업로드 진행률(ProgressBar) 표시. (완료)
    - 이미지/동영상 메시지 내 미리보기 (Inline Video Player 지원). (완료)
    - 파일 타입별 아이콘 및 다운로드 UI 최적화. (완료)

### 2.7 권한 및 보안 강화 (RBAC) (구현 완료 ✅)
- **현황**: 역할 기반 접근 제어(RBAC) 시스템 UI/UX 및 백엔드 동기화 완료.
- **필요 기능**:
    - 사용자 역할(Owner, Admin, Member)에 따른 관리 기능 노출 제어. (완료)
    - 팀/채널 삭제 및 수정 권한 제한. (완료)
    - 비권한 사용자의 관리 메뉴 접근 차단. (완료)

---

## 3. 향후 구현 로드맵
1. **Phase 1**: 팀 및 채널 관리 시스템 구축 (팀 생성, 멤버 관리 등)
2. **Phase 2**: 알림 시스템 및 스레드(답글) 기능 추가
3. **Phase 3**: 검색 기능 고도화 및 파일 프리뷰 UI 개선
4. **Phase 4**: 전반적인 보안 강화 및 UI 폴리싱
