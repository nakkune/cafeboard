# 문서화 스킬

## 설명
코드, API, 아키텍처에 대한 문서를 작성합니다.

## 지침

### 1. 문서 유형
- README: 프로젝트 개요
- API 문서: 엔드포인트 설명
- 코드 주석: 복잡한 로직 설명
- 설계 문서: 아키텍처 결정

### 2. API 문서 형식
각 엔드포인트에 대해 다음을 문서화:
```
### [METHOD] /api/endpoint
설명: 이 엔드포인트의 기능

파라미터:
| 이름 | 타입 | 필수 | 설명 |
|------|------|------|------|

응답:
- 성공 (200): 응답 설명
- 실패 (400/401/500): 에러 설명
```

### 3. 코드 주석 규칙
- 왜 이렇게 하는지: 설명 (무엇이 아닌)
- TODO/FIXME 주석 활용

### 4. 마크다운 규칙
- 제목은 # H1부터 순차 사용
- 표 활용하여 데이터 표현
- 코드 블록에 언어 지정
- 링크는 의미 있게

### 5. 작업 방법
1. 문서화할 대상 확인
2. 기존 문서 패턴 확인
3. 마크다운 형식으로 작성
4. docs/ 폴더에 저장 (해당하는 경우)

### 6. 저장 위치
- 프로젝트 문서: `docs/`
- API 문서: `docs/api.md`
- 설계 문서: `docs/design/`
