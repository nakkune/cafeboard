# 테스트 작성 스킬

## 설명
프론트엔드와 백엔드를 위한 테스트 코드를 작성합니다.

## 지침

### 1. 테스트 전략
- 유닛 테스트: 개별 함수/컴포넌트
- 통합 테스트: API 엔드포인트
- E2E 테스트: 사용자 시나리오

### 2. 프론트엔드 테스트
- Vitest/Jest 사용
- React Testing Library로 컴포넌트 테스트
- 필요한 경우 cypress로 E2E 테스트

### 3. 백엔드 테스트
- Jest 사용
- Supermock으로 API 테스트
- 테스트 데이터.fixture 패턴 사용

### 4. 테스트 작성 원칙
- AAA 패턴 적용 (Arrange, Act, Assert)
- 테스트 이름은 한국어로 설명
- 하나의 테스트에 하나의 확인
-.mock 또는 fixture로 테스트 데이터 관리

### 5. 작업 방법
1. 기존 테스트 패턴 확인 (동일한 유형의 파일 참조)
2. 테스트 파일 위치 결정
3. 테스트 코드 작성
4. 테스트 실행하여 확인

### 6. 파일 위치 규칙
- 프론트엔드: `src/__tests__/` 또는 `src/**/*.test.tsx`
- 백엔드: `src/**/*.test.ts`
