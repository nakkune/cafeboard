# 코드 리뷰 스킬

## 설명
프론트엔드와 백엔드 코드를 전문적으로 리뷰합니다.

## 지침

### 1. 코드 리뷰 원칙
- 코드 스타일과 일관성 확인
- 보안 취약점 식별
- 성능 문제 점검
- 버그 가능성 검토
- 모범 사례 적용 여부 확인

### 2. 프론트엔드 리뷰 시 확인사항
- React/Vue 적절한 사용
- 상태 관리 패턴 올바름
- 컴포넌트 구조 적절성
- 메모이제이션 적절 사용
- 접근성 고려
- 반응형 디자인 확인

### 3. 백엔드 리뷰 시 확인사항
- 보안 (SQL 인젝션, XSS 등)
- 에러 처리 적절성
- 데이터베이스 쿼리 최적화
- API 설계 일관성
- 인증/인가 로직 보안

### 4. 출력 형식
리뷰 결과는 다음 형식으로 출력:
- 파일 경로와 줄 번호
- 문제 유형 (Bug, Security, Performance, Style, Suggestion)
- 상세 설명
- 수정 제안

### 5. 작업 방법
1. 먼저 변경된 파일들을 확인 (git diff 또는 ls)
2. 각 파일을 읽고 분석
3. 발견된 문제를 파일:줄번호 형식으로 기록
4. 심각도 수준 할당 (Critical, High, Medium, Low)
5. 개선 제안 제공
